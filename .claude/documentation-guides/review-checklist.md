# 문서 리뷰 체크리스트

**문서 버전**: 1.0
**최종 수정일**: 2025-11-11
**상태**: Active

---

## 개요

문서 작성 완료 후 제출 전 반드시 확인해야 할 품질 기준을 정의합니다.

---

## 작성 완료 후 확인사항

### 1. 기본 원칙 준수

#### 한글 작성 원칙
- [ ] 한글로 작성되었는가? (CLAUDE.md 제외)
- [ ] 기술 용어는 영문 병기되었는가?
- [ ] 문장이 자연스러운가?

#### 간소화 원칙
- [ ] 문서가 200줄 이하인가?
- [ ] 200줄 초과 시 하위 문서로 분리되었는가?
- [ ] 하위 문서 링크가 명확한가?

#### 즉시 문서화 원칙
- [ ] 개발 완료 직후 작성되었는가?
- [ ] 코드와 함께 커밋 준비되었는가?
- [ ] 관련 문서들도 업데이트되었는가?

---

### 2. 내용 품질

#### 명확성
- [ ] 제목이 명확하고 구체적인가?
- [ ] 각 섹션의 목적이 분명한가?
- [ ] 전문 용어가 적절히 설명되었는가?

#### 완결성
- [ ] 필요한 모든 정보가 포함되었는가?
- [ ] 코드 예시가 포함되었는가?
- [ ] 실행 가능한 예시인가?

#### 정확성
- [ ] 코드 예시가 실제 동작하는가?
- [ ] 파일 경로가 정확한가?
- [ ] 버전 정보가 최신인가?

#### 연결성
- [ ] 관련 문서 링크가 있는가?
- [ ] 링크가 유효한가?
- [ ] 링크 텍스트가 명확한가?

---

### 3. 구조 및 가독성

#### 문서 구조
- [ ] 섹션이 논리적으로 구성되었는가?
- [ ] 계층 구조가 명확한가? (H1 → H2 → H3)
- [ ] 목차(TOC)가 있는가? (150줄 이상 문서)

#### 코드 블록
- [ ] 코드 블록에 언어가 명시되었는가?
- [ ] 코드에 주석이 포함되었는가?
- [ ] 코드가 올바르게 포맷되었는가?

#### 시각 자료
- [ ] 다이어그램이 적절히 사용되었는가?
- [ ] 표가 명확하게 작성되었는가?
- [ ] 스크린샷이 필요한 경우 포함되었는가?

#### 포맷팅
- [ ] 마크다운 문법이 올바른가?
- [ ] 줄바꿈이 적절한가?
- [ ] 강조 표시가 일관적인가? (볼드, 이탤릭)

---

### 4. 유지보수성

#### 메타데이터
- [ ] 문서 버전이 기록되었는가?
- [ ] 최종 수정일이 기록되었는가?
- [ ] 담당자/팀이 명시되었는가?
- [ ] 문서 상태가 명시되었는가? (Active, Deprecated 등)

#### 변경 이력
- [ ] 주요 변경사항이 기록되었는가?
- [ ] 변경 날짜가 명시되었는가?
- [ ] 변경 이유가 설명되었는가?

#### 리뷰 정보
- [ ] 리뷰 주기가 명시되었는가?
- [ ] 다음 리뷰 날짜가 설정되었는가?

---

## 문서 타입별 추가 체크리스트

### Feature Spec (기능 명세서)

- [ ] 기능 ID가 명시되었는가?
- [ ] 우선순위가 설정되었는가?
- [ ] 구현 상태가 표시되었는가?
- [ ] 사용자 스토리가 포함되었는가?
- [ ] 기술 명세가 충분한가?
- [ ] API 참조가 명확한가?
- [ ] 알려진 이슈가 기록되었는가?

### PRD (제품 요구사항)

- [ ] 제품 개요가 명확한가?
- [ ] 비즈니스 목표가 정의되었는가?
- [ ] 사용자 페르소나가 있는가?
- [ ] 기능 우선순위가 정의되었는가?
- [ ] 비기능 요구사항이 포함되었는가?

### API 문서

- [ ] 엔드포인트가 명확한가?
- [ ] Request/Response 예시가 있는가?
- [ ] 에러 코드가 정의되었는가?
- [ ] 인증 방법이 설명되었는가?

---

## 리뷰 프로세스

### 1단계: 자가 리뷰
위 체크리스트 항목 모두 확인

### 2단계: 동료 리뷰
- 다른 팀원에게 리뷰 요청
- 피드백 반영

### 3단계: 최종 검증
- 모든 링크 동작 확인
- 코드 예시 실행 확인
- 문서 길이 확인 (200줄 이하)
